# 千问 App 需求文档

**版本**: v1.0  
**日期**: 2026-06-11  
**项目**: 基于通义千问 API 的 AI 对话应用

---

## 1. 项目概述

构建一个基于阿里云通义千问（Qwen）系列模型的 Web 应用，支持多轮对话、多模态输入、模型动态切换，以及结构化卡片内容展示。

---

## 2. 功能模块

### 2.1 基础 Chat 能力

**目标**: 提供流畅的多轮对话体验。

| 需求 | 描述 |
|------|------|
| 多轮上下文 | 维护完整的会话历史，支持上下文感知的连续对话 |
| 流式输出 | 使用 SSE/Stream 模式逐字输出回复，减少等待感 |
| 消息气泡 | 区分用户消息与 AI 回复，时间戳展示 |
| 会话管理 | 新建会话、切换历史会话、删除会话 |
| Markdown 渲染 | 支持代码块、列表、表格、加粗等 Markdown 格式渲染 |
| 代码高亮 | 对代码块进行语言识别与语法高亮 |
| 停止生成 | 支持中途停止流式输出 |
| 重新生成 | 对最后一条回复支持重新生成 |

| 一键复制粘贴 | 对模型输出的内容复制到剪切板 |


---

### 2.2 多模态能力

**目标**: 支持图文混合输入，充分利用 Qwen-VL 视觉理解能力。

| 需求 | 描述 |
|------|------|
| 图片上传 | 支持本地图片上传（JPG/PNG/WEBP/GIF），大小限制 20MB |
| 图片预览 | 上传后在输入框上方展示缩略图，可删除 |
| 图文问答 | 将图片与文字一并发送给视觉模型进行理解与回答 |
| URL 图片 | 支持粘贴图片 URL 进行多模态分析 |
| 文件上传（扩展）| 支持 PDF/Word 文档上传，提取文本后发送（可选） |

**对应模型**: `qwen-vl-plus`、`qwen-vl-max`

---

### 2.3 模型切换

**目标**: 允许用户在不同 Qwen 模型间自由切换，适配不同场景需求。

#### 支持的模型列表

| 模型 ID | 显示名称 | 特点 |
|---------|----------|------|
| `qwen-turbo` | Qwen Turbo | 响应快，成本低，适合日常对话 |
| `qwen-plus` | Qwen Plus | 均衡能力，推荐默认 |
| `qwen-max` | Qwen Max | 最强推理，适合复杂任务 |
| `qwen-vl-plus` | Qwen VL Plus | 视觉理解，中等质量 |
| `qwen-vl-max` | Qwen VL Max | 视觉理解，最高质量 |
| `qwen-long` | Qwen Long | 超长上下文（1M tokens） |

#### 切换交互

- 顶部或侧边栏提供模型选择下拉框
- 切换模型时提示"是否清空当前会话"（部分模型不兼容历史格式）
- 当前选中模型始终可见
- 视觉模型仅在有图片输入时自动推荐或强制切换

---

### 2.4 卡片展示

**目标**: 将结构化的 AI 回复以视觉友好的卡片形式呈现，提升信息可读性。

#### 卡片类型

| 卡片类型 | 触发场景 | 展示内容 |
|----------|----------|----------|
| **代码卡片** | 回复包含代码块 | 语言标签、代码内容、一键复制按钮 |
| **搜索结果卡片** | 涉及信息检索 | 标题、摘要、来源链接 |
| **步骤卡片** | 回复包含有序步骤 | 数字序号、步骤标题、步骤描述 |
| **对比卡片** | 回复涉及多项对比 | 多列对比表格，高亮差异 |
| **天气/数据卡片** | 结构化数据回复 | 图标 + 数值，可视化展示 |
| **图片分析卡片** | 多模态分析结果 | 缩略图 + 分析文字并排显示 |

#### 卡片交互

- 卡片可折叠/展开
- 代码卡片支持全屏查看
- 卡片内容支持单独复制

---

## 5. API 接入

使用阿里云 DashScope，兼容 OpenAI 格式：

```
Base URL: https://dashscope.aliyuncs.com/compatible-mode/v1
API Key: 环境变量 DASHSCOPE_API_KEY
```

- 文本对话：`POST /chat/completions`（stream: true）
- 视觉理解：消息中包含 `image_url` 类型的 content
- 模型参数：`temperature`、`max_tokens`、`top_p` 可在设置中调整

## 6. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 首屏加载 < 2s，流式首字节 < 1s |
| 安全 | API Key 不暴露在前端，通过后端代理转发 |
| 响应式 | 适配桌面端（1280px+）和移动端（375px+） |
| 兼容性 | Chrome 90+、Firefox 88+、Safari 14+ |
